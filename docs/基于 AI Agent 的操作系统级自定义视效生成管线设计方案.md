# 基于 AI Agent 的操作系统级自定义视效生成管线设计方案

## 一、 问题背景与系统定位

当前操作系统视窗渲染（如 Android/iOS）的高阶动态视效面临预制图形管线表达力不足的问题。传统依赖图形工程师将 UX 设计稿人工转化为底层 Shader 代码的过程成本高、容错低且与系统耦合。

本系统旨在构建一套**面向视觉特征的自动化闭环编译管线**，通过引入多模态大模型与物理约束（Harness），实现从“UX 设计意图”到“沙盒化高性能图形算子”的无人工干预转化。

------

## 二、 系统整体输入输出（I/O）矩阵

| **阶段**   | **模块**       | **输入 (Input)**                                             | **输出 (Output)**                                       | **约束条件**                                                 |
| ---------- | -------------- | ------------------------------------------------------------ | ------------------------------------------------------- | ------------------------------------------------------------ |
| **编辑态** | **生成 Agent** | 视觉参考：定频视频流 (MP4/WebM) 或 图像序列。 逻辑参考：动效曲线 (Lottie JSON/Figma 参数)。 | 视效中间表示 (DSL/JSON)。                               | 视频必须包含完整的时域周期（Loop）；禁止直接输入文字描述（避免二义性）。 |
| **编辑态** | **验证 Agent** | 候选 DSL、标准 UX 输入源。                                   | 反馈修正指令（如参数微调幅度）或 阻断报错（算力超标）。 | 强制多维特征评估（形态、时域、频域、性能）。                 |
| **运行时** | **编译器**     | 视效中间表示 (DSL/JSON)。                                    | 目标平台着色器代码 (GLSL/MSL/Warp)。                    | 静态语法与边界检查（禁用动态循环分支）。                     |

### 架构节点说明

- **隔离界限（Boundary）**：双线 `===` 划分了离线环境与设备端环境。编辑态运行在高算力的云端或开发者工作站，承担大模型推理与模拟压测；运行态部署在终端设备（如手机、MR 眼镜），仅保留轻量级转译与渲染能力，确保系统精简。
- **数据流转核心（DSL）**：跨越编辑态与运行态的唯一媒介是无执行风险的描述性文本（DSL）。大模型不直接向系统环境投递可执行代码，从架构物理层面上切断了 AI 幻觉导致的系统级崩溃风险。
- **约束回路（The Harness Loop）**：生成节点与感知/约束节点构成对抗或合作网络。所有视觉与性能约束指标均被量化，未通过审计的生成结果被严格阻断在编辑态内部。

## 四、 编辑态 (Editor State)：闭环演进系统设计

编辑态是核心的 AI 生成与迭代环境，其本质是一个**基于特征空间对齐的自动化寻优引擎**。

### 1. UX 输入约束与特征提取

为降低不确定性，系统要求强格式输入：

- **输入规范**：UX 设计需提供标准的 `60fps` 或 `120fps` 渲染视频，辅以关键帧的物理参数（如弹簧阻尼系数）。
- **特征提取 (Perception)**：多模态感知模块将输入视频降维为特征张量：
  - **形态场**：提取关键帧的有向距离场 (SDF) 轮廓。
  - **光流场**：计算帧间的光流矢量 (Optical Flow)，映射时域运动。
  - **色彩频域**：提取色彩直方图及空间频率分布（识别平滑过渡或高频噪点）。

### 2. 算子抽象与视效描述 (DSL)

禁止 AI 直接输出底层 C-style 代码，使用严格封装的领域专用语言 (DSL)。

- **基础算子库 (Skill Pack)**：包含 `SDF_Box`, `SDF_Sphere`, `Fractal_Noise`, `Gaussian_Blur`, `Fresnel_Schlick` 等纯数学函数。
- **场景描述文件 (IR)**：系统最终收敛的产物。描述算子的组合拓扑、Uniforms 映射及混合模式。

### 3. 约束迭代系统 (The Harness Loop)：自我迭代路径

Agent 通过验证器的反馈，在沙盒中进行“生成-采样-对比-修正”的闭环迭代，逼近 UX 设计。

- **初始化 (Stage 0)**：基于感知特征，检索预设算子模板（如识别到“毛玻璃”特征，提取 `Backdrop_Blur` + `Luminance_Noise` 模板）。
- **形态收敛 (Shape Phase)**：将当前 DSL 渲染为掩码图 (Mask)。计算与 UX 设计关键帧掩码的 IoU (交并比)。若 $IoU < 0.95$，反馈模块生成指令：“调整 `SDF` 圆角参数与布尔求并逻辑”。
- **动态拟合 (Motion Phase)**：在 $t \in [0, 1]$ 周期内连续采样渲染结果的光流。对比生成光流与 UX 光流的均方误差 (MSE)。若存在相位差，反馈模块指令：“将时间变量 $t$ 的驱动函数由线性 `fract(t)` 修改为 `smoothstep` 平滑过渡”。
- **性能剪枝 (Performance Phase)**：对收敛的 DSL 提取 AST（抽象语法树），统计指令复杂度（Texture Fetch 次数、ALU 运算量）。若测算 GPU Cycle 超出阈值，强制 Agent 进行算子降级。

## 五、 运行时 (Runtime State)：沙盒化执行环境

运行时系统负责将编辑态生成的结构化描述部署至设备，实现系统级解耦与安全执行。

### 1. 运行时 I/O 与上下文

- **输入通信 (Uniforms Injection)**：系统侧向视效沙盒高频注入标准化参数。
  - `u_time`: 全局时间戳，驱动周期性动效。
  - `u_resolution`: 视窗分辨率，维持 SDF 比例。
  - `u_pointer`: 触摸/光标的三维坐标 $(x, y, pressure)$，触发涟漪、形变等交互逻辑。
  - `u_sysTheme`: 系统主题（Dark/Light）的色调映射矩阵。
- **执行拓扑**：DSL 转译后的 Shader 挂载在系统 UI 树的特定层级（如背景层、Overlay 层），仅对限定的 Framebuffer 进行着色，不干预 DOM/View 树排版。

### 2. 物理边界与安全性

- **动态编译与缓存**：利用目标图形 API (Vulkan Pipeline Cache / Metal Library) 在后台预编译 DSL 转译的 Shader，确保首次渲染零延迟。
- **受限计算图 (Restricted Compute Graph)**：屏蔽所有导致内存越界的 API（如无界 Buffer 写入）。算子均保证数学域连续，杜绝除零或无穷大 (NaN) 引起的 GPU 重置。

## 六、 核心差异：本系统 vs. A2UI

本系统关注底层图形计算的数学表达与物理安全，A2UI 关注上层逻辑组件的语义排版。

| **评估维度**         | **A2UI (Agent to UI)**                                 | **自定义视效 Agent 系统 (Visual FX)**                        |
| -------------------- | ------------------------------------------------------ | ------------------------------------------------------------ |
| **处理对象与维度**   | 结构化组件树（Button, List），2D 布局 $(x, y, w, h)$。 | 算子逻辑与片段着色，高维时空场 $(RGB, \alpha, uv, t, depth)$。 |
| **特征提取与目标**   | 语义解析，交互逻辑准确率。                             | 光流、距离场、频域映射，时空视觉一致性。                     |
| **底层硬件负载**     | CPU 密集型（DOM/View 树解析与约束排版计算）。          | GPU 密集型（大规模并行像素着色、张量计算）。                 |
| **系统崩溃风险**     | 应用级（布局错乱、视图重叠、点击失效）。               | 系统级（着色器死循环导致 GPU 挂起、发热降频、帧率骤降）。    |
| **验证迭代阻断条件** | 语义不匹配、组件丢失。                                 | GPU 指令超载、动态频率闪烁、SDF 不连续。                     |
| **解耦层级**         | 视图展示与数据状态层的逻辑解耦。                       | 高阶图形算子与系统级核心渲染管线的物理隔离。                 |
